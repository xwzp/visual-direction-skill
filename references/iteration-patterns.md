# 常见 Delta Update 模式

这是迭代模式使用的参考。当用户拿 AI 生成结果回来说"不对"时，先**诊断是哪一类不协调**，然后套用对应的模式。

每个模式都给出「症状 → 诊断 → 干预手法」三段。

**平台通用性**：这些模式在网站、移动 App、桌面应用都适用。具体到动效参数时，注意切换到对应平台的语法（web 用 Framer Motion spring 参数、iOS 用 SwiftUI Animation、Android 用 Material spring，具体见 `platform-conventions.md`）。以下例子以 web 为主，其他平台按比例转换。

---

## 模式 1：Signature moment 做出来了，但周围元素显得孤立

### 症状
Hero 标题（或某个主要元素）做得很好（彩色、大字、有趣的排版），但下方的 chips、logo、辅助元素看起来"死气沉沉"、像"排排站的士兵"，整体不协调。

### 诊断
不是周围元素"不够有趣"，是它们**没有从 signature moment 继承视觉系统**。signature moment 建立了一套色彩家族和一套形变语汇（例如旋转），但其他元素还在用旧系统。

### 干预手法：让其他元素"认领"signature moment 的元素

**具体做法**：
- 识别 signature moment 用了哪几个颜色（通常 2-4 个）
- 给周围 1-3 个辅助元素**各自分配一个颜色**（不要全部同色，要让它们互相区分）
- 给这些元素加上**更小幅度**的形变（例如 hero 标题旋转了 ±4°，chips 只旋转 ±1-2°）
- 加上**idle 动画**（极慢的呼吸/漂浮），周期各不相同，让它们像"在自己呼吸"而不是"同步摆动"

**关键约束**：
- 不要让功能性元素（主输入框、主 CTA、正文）也变彩色——它们应该保持稳定
- 不要把旋转角度加到超过 ±3°，否则从"有活力"变成"故障感"
- 多个元素的 idle 动画周期要故意错开（例如 3s、4s、5s），不能同步

### 核心原则引用
原则 6：继承而非照搬。

## 模式 2：核心内容区（产品的"主角"）显得平淡

### 症状
页面其他部分做得都不错，但最重要的内容展示区（对命名工具来说是"生成的名字"，对博客来说是"文章标题和封面"，对作品集来说是"作品本身"）显得普通。

### 诊断
产品的"主角内容"没有被当作主角对待。字号可能和次要文字接近，颜色可能和其他元素相同，没有专属的视觉处理。

### 干预手法：让主角大胆起来

**具体做法**：
- **字号**：主角内容的字号应该是全站**第二大**的（仅次于 hero 标题）。如果 hero 是 80px，主角内容应该 48-72px，不能低于 40px。
- **颜色渗透**：如果页面已经有 accent 色系统，让主角内容区**继承**对应的 accent 色（通常通过背景渐变 6-10% opacity、left accent bar、或辅助文字着色的方式）
- **入场动画**：主角内容出现时用 spring，stagger 100-120ms 逐个落地（不要 fade-in，用 spring + 轻微 overshoot）
- **hover 交互**：主角应该是可以 hover 触发额外信息的——hover 后展开一行小字解释/metadata

**字号建议**：
- Hero 标题：56-80px
- **主角内容：48-72px**（新手常错地做成 32-40px）
- 次级标题：28-40px
- 正文：15-18px
- Metadata / 辅助：12-13px

### 核心原则引用
原则 5：签名时刻 + 稳定支点。把预算砸到主角上。

## 模式 3："随机倾斜" 看起来很机械

### 症状
设计里想要"歪七扭八有活力"的效果，但看起来每次加载都一样、几张卡片的旋转组合总是那样——不像真的活的，像"设计师摆拍的活泼"。

### 诊断
实现时用了**固定**的旋转值（卡 A 永远 -2°、卡 B 永远 +1.5°），不是**运行时随机**。

### 干预手法：真·随机

**具体做法**：
- 每张卡片挂载时**在一个范围内随机**生成旋转角度
- 范围建议：`-2.5° 到 +2.5°`，但**排除 -0.5° 到 +0.5°`**（接近 0 看起来像没转，反而不好）
- 颜色映射**保持固定**（不要随机），只旋转随机

**伪代码**：
```javascript
const getRandomTilt = () => {
  const ranges = [[-2.5, -0.5], [0.5, 2.5]];
  const range = ranges[Math.floor(Math.random() * 2)];
  return Math.random() * (range[1] - range[0]) + range[0];
};
```

**重要约束**：
- 同一 session 内**不要重新随机**（用户点 Next 又 Back 回来，角度应该保持一致，否则会晕）
- 颜色是信息（传达分类），旋转是气质（传达活力），两者处理逻辑不同

## 模式 4：辅助元素（进度条、小 logo）和主元素气质不匹配

### 症状
主标题 / 主内容做得很大很有戏，但页面顶部的小 logo、进度条、nav 看起来"小得很沉闷"，和下面的活泼反差太大。

### 诊断
辅助元素被当作"功能性元素"设计了（小、中性、不出戏），但当主元素变得很戏剧时，辅助元素需要也**略微升级**以匹配能量等级。注意是"略微"而不是"同等"——它们仍然是辅助，但不能"太辅助"。

### 干预手法：按比例升级辅助元素

**具体做法**：

**如果是 logo 显得太小：**
- 增加 logo 尺寸（按比例到主标题字号的 60-80%）
- 加上 idle 动画（慢速旋转 / 呼吸 / 脉冲）
- 考虑让 logo 本身使用**主标题的颜色系统**（例如渐变填充，从紫到粉到青）
- 去掉会让 logo 显得"被框住"的容器（例如 iOS app icon 式的 squircle 背景）

**如果是进度条显得太细：**
- 厚度从 1px → 3px
- 长度横跨整个内容区宽度
- 填充色从单色 → **多色渐变**（直接继承主标题的颜色系统）
- 填充动画用 spring（`stiffness: 200, damping: 30`，比标准 spring 慢一点）

**核心原则引用**
原则 6：继承而非照搬。辅助元素从主元素继承色彩系统，但用更克制的方式（更小字号、更轻动效）。

## 模式 5：副本 chips / tags 单调

### 症状
示例标签、category chips 等小元素都是一样的 outline + 一样的字体 + 一样的字号，看起来像"同一个模子印的"。

### 诊断
没有利用 accent 系统给它们差异化。chips 天然适合承载色彩身份（它们是离散的、并列的，彼此有差异是合理的）。

### 干预手法

**具体做法**：
- 三个/四个 chips 各自"认领"accent 色家族中的一个颜色
- Border 和文字同色（保持 outline 样式，背景透明）
- Border 宽度从 1px → **1.5px**，让颜色更显眼
- 字号略增（+10-15%）
- 每个 chip 加一个极轻微的旋转（±2°），角度各不相同
- **hover 时**：rotate 回到 0°（"站直"）+ scale 1.05 + 填充对应色 10% opacity 背景

### 核心原则引用
原则 6：继承而非照搬。chips 继承主色系，但作为离散元素，每个认领不同颜色。

## 模式 6：AI 生成的整体方向就是错的（例如奶油底但你想要暗色）

### 症状
拿到 AI 输出一看，整个色系、字体、气质都和你想的完全不是一回事。

### 诊断
这不是 delta 的问题，是 **brief 本身带错方向了**。大概率你的 brief 里包含了把 AI 推向错误方向的**方向词**。

### 干预手法：不要做 delta，去重写 brief

**具体做法**：
1. 回到 brief，找到那些方向词（见 `brief-template.md` 里的方向词对照表）
2. 把错误方向的词全部删掉
3. 换成正确方向的参考产品名（"像 Raycast 而不是像 Notion"）
4. **大幅扩充 "❌ 不要做" 清单**，明确禁止错误方向的所有语汇
5. 在结尾加一句话总结，再次强调正确方向

**判断标准**：如果要改动的地方超过 2-3 处，就不是 delta 的活，是重写 brief。

## Delta Update 的标准格式

所有 delta 都应该遵循这个结构：

```markdown
# Delta Update v[N] — [本次要改的范围]

保持 [未被改动的部分] 不变。
本次只修改 [具体目标]。

## 诊断
[症状 + 为什么不对]

## 设计原则
[这次改动遵循的原则，通常引用某一条核心原则]

## 新规格
[具体的数值、颜色、动效参数]

## ❌ 继续不要做的
[重申禁止项，防止 AI 借机失控]

## 一句话核心
[用一句话概括本次改动的精神]
```

**❌ 继续不要做的** 这一节很关键——AI 在 delta 时会觉得"用户让我改这里，说明之前的约束可以松一松"，然后就偷偷破坏了其他地方。必须重申禁止项。
